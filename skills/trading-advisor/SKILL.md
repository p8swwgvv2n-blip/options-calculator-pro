`分析`、`查询`、`股票的操作建议`、`持仓分析`、`看看我的股票`、`期权分析`、`portfolio`、`analyze`

## 工作流程

### 1. 读取持仓数据

**首先**读取用户的持仓记忆文件（位于工作区或用户指定位置）。

文件命名规则（按优先级尝试）：
- `portfolio.json` / `持仓.json` / `我的持仓.json`
- 如果找不到，询问用户文件路径

**持仓数据格式**：

```json
{
  "positions": [
    {
      "ticker": "AAPL",
      "type": "stock",
      "shares": 100,
      "avgCost": 180.50,
      "addedDate": "2024-01-15"
    },
    {
      "ticker": "TSLA",
      "type": "option",
      "optionType": "call",
      "strike": 200,
      "expiry": "2024-06-21",
      "contracts": 2,
      "avgCost": 15.30,
      "addedDate": "2024-03-10"
```

如果用户没有持仓文件，询问：
1. 要分析哪些股票/期权？
2. 当前持仓情况（数量、成本、到期日等）

### 2. 逐个持仓分析

对每个持仓执行以下分析：

#### A. 股票类持仓

**技术指标**（通过 Twelve Data / Alpha Vantage API 获取历史数据后计算）：

| 指标 | 参数 | 判断逻辑 |
|------|------|----------|
| **MACD** | (12, 26, 9) | 金叉→多，死叉→空；DIF/DEA 距离判断动能 |
| **KDJ** | (9, 3, 3) | K>80 超买，K<20 超卖；金叉/死叉信号 |
| **布林带** | (20, 2σ) | 价格在上轨附近→压力，下轨→支撑；带宽判断波动 |
| **成交量** | 20日均量 | 放量上涨→强，缩量→弱；量价配合度 |

**计算公式**：
- MACD: EMA12 - EMA26, Signal = EMA9(MACD)
- KDJ: RSV = (C-L9)/(H9-L9)*100, K = 2/3*prev_K + 1/3*RSV
- Bollinger: MA20 ± 2*STD20
- Volume Ratio: 当日成交量 / 20日均量

**综合评级**：
- 强多：MACD金叉 + KDJ上升 + 布林中轨上方 + 放量
- 中性偏多：MACD零轴上 + KDJ中性 + 布林中轨附近
- 观望：MACD死叉 + KDJ超买后回落 + 缩量
- 强空：MACD死叉 + KDJ超卖 + 布林下轨下方 + 放量下跌

#### B. 期权类持仓

**IV 分析**：
- 当前 IV 百分位（vs 过去 30/60/90 天）
- IV vs HV20 对比（高估/低估判断）
- IV 趋势（上升/下降/平稳）
- 距离到期日的 Theta 衰减速度

**Greeks 快照**：
- Delta: 方向敏感度
- Theta: 每日时间损耗
- Vega: IV 变动 1% 的影响
- Gamma: Delta 变化速度

**关键判断**：
- 剩余天数 ≤ 7 天 → 考虑平仓（时间损耗加速）
- IV 高于历史 75 分位 → 卖方有利，考虑止盈
- IV 低于历史 25 分位 → 买方有利，持有或加仓

### 3. 消息面分析

**使用 WebSearch 工具**搜索以下内容（每个持仓单独搜索）：

1. **分析师评级变化**（最近 7 天）
   - 搜索：`{ticker} analyst rating upgrade downgrade 2026`
   - 提取：买入/卖出评级变化、目标价调整

2. **财报与重大事件**
   - 搜索：`{ticker} earnings report news 2026`
   - 提取：财报日期、预期、近期公告

3. **行业动态**
   - 搜索：`{ticker} sector industry news analysis`
   - 提取：行业政策、竞争格局变化

4. **市场情绪**
   - 搜索：`{ticker} options unusual activity sentiment`
   - 提取：期权市场异动、Put/Call 比率

**消息面评级**：
- 利好：多家机构上调评级、超预期财报、行业利好
- 中性：消息混杂、无重大变化
- 利空：评级下调、财报不及预期、监管风险

### 4. 应用交易规则

#### 期权类规则

```
到期日预留：距离到期 ≤ 7 天 → 建议平仓
最大持仓：同一标的 ≤ 4 张合约
止盈规则：盈利 ≥ 100% → 建议止盈
止损规则：亏损 ≥ 50% → 建议止损
```

**操作建议生成**：
- 剩余 ≤ 7 天 + 盈利 → "建议平仓止盈，时间损耗加速"
- 剩余 ≤ 7 天 + 亏损 → "建议平仓止损，避免归零风险"
- 盈利 ≥ 100% → "已达到 100% 止盈线，建议分批止盈（先平一半）"
- 持仓 > 4 张 → "超出最大持仓限制，建议减仓至 4 张以内"

#### 股票类规则

```
止盈一档：盈利 ≥ 5% → 卖出 1/3 仓位
止盈二档：盈利 ≥ 10% → 再卖出 1/3
移动止盈：盈利回撤 ≥ 3% → 触发剩余仓位止盈
止损规则：亏损 ≥ 8% → 建议止损
```

**移动止盈计算**：
- 记录最高盈利点位
- 当前价格从最高点回撤 ≥ 3% → 触发止盈
- 公式：trigger = highestPrice * 0.97

**操作建议生成**：
- 盈利 5-10% → "第一档止盈：建议卖出 1/3 仓位锁定利润"
- 盈利 > 10% → "第二档止盈：建议再卖出 1/3"
- 盈利回撤 > 3% → "移动止盈触发：从最高点 {highest} 回撤 {drawdown}%，建议平仓剩余"
- 亏损 > 8% → "触及止损线，建议止损"

### 5. 仓位管理与资金分配

#### A. 仓位健康度评估

**集中度检查**：
```
单只股票占比 = (当前价 × 股数) / 总组合价值
单只期权占比 = (当前价 × 合约数 × 100) / 总组合价值
```

| 集中度 | 评级 | 建议 |
|--------|------|------|
| 单只 > 30% | ⚠️ 过高 | 建议减仓至 20% 以下，分散风险 |
| 单只 20-30% | ⚡ 偏高 | 可接受但需关注，不建议加仓 |
| 单只 10-20% | ✅ 合理 | 正常持仓 |
| 单只 < 10% | ℹ️ 偏低 | 如看好可适量加仓 |

**行业集中度**：
```
同行业持仓 = 同板块所有持仓之和
同行业占比 = 同行业持仓 / 总组合价值
```
- 同行业 > 50% → "行业集中度过高，建议分散至其他板块"
- 同行业 > 30% → "行业占比较高，注意板块风险"

**相关性检查**：
- 持仓中相关性 > 0.7 的标的视为"同质化"
- 同质化标的合计占比 > 40% → "组合同质化严重，建议增加低相关品种"

#### B. 现金管理

**现金比例规则**：
```
现金占比 = 可用现金 / 总组合价值
```

| 现金占比 | 评级 | 建议 |
|----------|------|------|
| < 10% | ⚠️ 过满 | 仓位过重，建议减仓或止盈部分持仓释放现金 |
| 10-20% | ✅ 合理 | 保持适度现金应对波动 |
| 20-35% | ⚡ 偏高 | 现金闲置，可考虑建仓观察名单标的 |
| > 35% | ⚠️ 空仓 | 现金过多，考虑加仓或配置货币基金 |

**建议操作**：
- 现金 < 10% → 列出可止盈的持仓，建议减仓释放现金
- 现金 > 30% → 列出观察名单中技术面较好的标的，建议分批建仓

#### C. 新建仓建议

**单笔建仓金额**（固定比例法）：
```
单笔风险预算 = 总组合价值 × 1-2%
建仓金额 = 单笔风险预算 / 止损比例

示例：组合 $50,000，单笔风险 1% = $500
如果止损设 5%，则建仓金额 = $500 / 5% = $10,000（即 20% 仓位）
```

**分批建仓计划**：
```
首次建仓：计划仓位的 1/3
确认加仓：技术面确认后加 1/3
回调补仓：回调至支撑位后补最后 1/3
```

**期权建仓限制**：
- 单笔期权权利金 ≤ 总组合价值的 5%
- 单标的最多 4 张合约
- 到期日 ≥ 30 天（避免短期 Theta 损耗）

#### D. 再平衡建议

**触发条件**：
- 任一持仓偏离目标仓位 > 5 个百分点
- 每季度定期再平衡
- 重大事件后（财报、拆股等）

**再平衡操作**：
```
目标仓位 = 各持仓目标占比 × 总组合价值
调仓量 = 目标仓位 - 当前仓位

正数 → 加仓
负数 → 减仓
```

**输出示例**：
```
【仓位管理】 综合评级: {健康度}
┌─────────────────────────────────────────
│ 组合总值: ${totalValue}
│ 现金: ${cash} ({cashPct}%) — {现金评级}
│ 持仓集中度: 最高 {topTicker} {topPct}% — {集中度评级}
│ 行业分布: {sectorBreakdown}
│ 同质化标的: {correlatedPairs}
├─────────────────────────────────────────
│ 📋 调仓建议:
│ {ticker1}: 当前 {currentPct1}% → 目标 {targetPct1}% ({action1} {amount1})
│ {ticker2}: 当前 {currentPct2}% → 目标 {targetPct2}% ({action2} {amount2})
│ 现金: 当前 {cashPct}% → 目标 {targetCash}%
└─────────────────────────────────────────
```

#### E. 风险预算分配

**组合层面风险**：
```
最大回撤容忍 = 总组合价值 × 10%
单只最大亏损 = 总组合价值 × 2%
期权最大亏损 = 权利金归零
```

**风险分配**：
- 股票类：总风险预算的 60-70%
- 期权类：总风险预算的 20-30%
- 现金/固收：总风险预算的 10-20%

**相关性调整**：
- 高相关持仓（ρ > 0.7）：共享风险预算
- 低相关持仓（ρ < 0.3）：独立风险预算
- 负相关持仓（ρ < 0）：可对冲，降低总风险预算占用

## 注意事项

1. **免责声明**：每次输出末尾附加「以上分析仅供学习参考，不构成投资建议」
2. **数据延迟**：免费 API 有 15 分钟延迟，提醒用户注意
3. **期权数据限制**：如果无法获取实时期权 IV，使用 HV20 估算并明确标注
4. **消息面时效性**：WebSearch 结果可能滞后，建议结合券商实时资讯
5. **持仓更新**：分析完成后提示用户更新持仓记忆文件
