### 2. 逐个持仓分析

对每个持仓执行以下分析：

#### A. 股票类持仓

**技术指标**（通过 Twelve Data / Alpha Vantage API 获取历史数据后计算）：

| 指标 | 参数 | 判断逻辑 |
|------|------|----------|
| **MACD** | (12, 26, 9) | 金叉→多，死叉→空；DIF/DEA 距离判断动能 |
| **KDJ** | (9, 3, 3) | K>80 超买，K<20 超卖；金叉/死叉信号 |
| **布林带** | (20, 2σ) | 价格在上轨附近→压力，下轨→支撑；带宽判断波动 |
| **成交量** | 20日均量 | 放量上涨→强，缩量→弱；量价配合度 |

**计算公式**：
- MACD: EMA12 - EMA26, Signal = EMA9(MACD)
- KDJ: RSV = (C-L9)/(H9-L9)*100, K = 2/3*prev_K + 1/3*RSV
- Bollinger: MA20 ± 2*STD20
- Volume Ratio: 当日成交量 / 20日均量

**综合评级**：
- 强多：MACD金叉 + KDJ上升 + 布林中轨上方 + 放量
- 中性偏多：MACD零轴上 + KDJ中性 + 布林中轨附近
- 观望：MACD死叉 + KDJ超买后回落 + 缩量
- 强空：MACD死叉 + KDJ超卖 + 布林下轨下方 + 放量下跌

#### B. 期权类持仓

**IV 分析**：
- 当前 IV 百分位（vs 过去 30/60/90 天）
- IV vs HV20 对比（高估/低估判断）
- IV 趋势（上升/下降/平稳）
- 距离到期日的 Theta 衰减速度

**Greeks 快照**：
- Delta: 方向敏感度
- Theta: 每日时间损耗
- Vega: IV 变动 1% 的影响
- Gamma: Delta 变化速度

**关键判断**：
- 剩余天数 ≤ 7 天 → 考虑平仓（时间损耗加速）
- IV 高于历史 75 分位 → 卖方有利，考虑止盈
- IV 低于历史 25 分位 → 买方有利，持有或加仓

### 3. 消息面分析

**使用 WebSearch 工具**搜索以下内容（每个持仓单独搜索）：

1. **分析师评级变化**（最近 7 天）
   - 搜索：`{ticker} analyst rating upgrade downgrade 2026`
   - 提取：买入/卖出评级变化、目标价调整

2. **财报与重大事件**
   - 搜索：`{ticker} earnings report news 2026`
   - 提取：财报日期、预期、近期公告

3. **行业动态**
   - 搜索：`{ticker} sector industry news analysis`
   - 提取：行业政策、竞争格局变化

4. **市场情绪**
   - 搜索：`{ticker} options unusual activity sentiment`
   - 提取：期权市场异动、Put/Call 比率

**消息面评级**：
- 利好：多家机构上调评级、超预期财报、行业利好
- 中性：消息混杂、无重大变化
- 利空：评级下调、财报不及预期、监管风险

### 4. 应用交易规则

#### 期权类规则

```
到期日预留：距离到期 ≤ 7 天 → 建议平仓
最大持仓：同一标的 ≤ 4 张合约
止盈规则：盈利 ≥ 100% → 建议止盈
止损规则：亏损 ≥ 50% → 建议止损
```

**操作建议生成**：
- 剩余 ≤ 7 天 + 盈利 → "建议平仓止盈，时间损耗加速"
- 剩余 ≤ 7 天 + 亏损 → "建议平仓止损，避免归零风险"
- 盈利 ≥ 100% → "已达到 100% 止盈线，建议分批止盈（先平一半）"
- 持仓 > 4 张 → "超出最大持仓限制，建议减仓至 4 张以内"

#### 股票类规则

```
止盈一档：盈利 ≥ 5% → 卖出 1/3 仓位
止盈二档：盈利 ≥ 10% → 再卖出 1/3
移动止盈：盈利回撤 ≥ 3% → 触发剩余仓位止盈
止损规则：亏损 ≥ 8% → 建议止损
```

**移动止盈计算**：
- 记录最高盈利点位
- 当前价格从最高点回撤 ≥ 3% → 触发止盈
- 公式：trigger = highestPrice * 0.97

**操作建议生成**：
- 盈利 5-10% → "第一档止盈：建议卖出 1/3 仓位锁定利润"
- 盈利 > 10% → "第二档止盈：建议再卖出 1/3"
- 盈利回撤 > 3% → "移动止盈触发：从最高点 {highest} 回撤 {drawdown}%，建议平仓剩余"
- 亏损 > 8% → "触及止损线，建议止损"

### 5. 仓位管理与资金分配

#### A. 仓位健康度评估

**集中度检查**：
```
单只股票占比 = (当前价 × 股数) / 总组合价值
单只期权占比 = (当前价 × 合约数 × 100) / 总组合价值
```

| 集中度 | 评级 | 建议 |
|--------|------|------|
| 单只 > 30% | ⚠️ 过高 | 建议减仓至 20% 以下，分散风险 |
| 单只 20-30% | ⚡ 偏高 | 可接受但需关注，不建议加仓 |
| 单只 10-20% | ✅ 合理 | 正常持仓 |
| 单只 < 10% | ℹ️ 偏低 | 如看好可适量加仓 |

**行业集中度**：
```
同行业持仓 = 同板块所有持仓之和
同行业占比 = 同行业持仓 / 总组合价值
```
- 同行业 > 50% → "行业集中度过高，建议分散至其他板块"
- 同行业 > 30% → "行业占比较高，注意板块风险"

**相关性检查**：
- 持仓中相关性 > 0.7 的标的视为"同质化"
- 同质化标的合计占比 > 40% → "组合同质化严重，建议增加低相关品种"

#### B. 现金管理

**现金比例规则**：
```
现金占比 = 可用现金 / 总组合价值
```

| 现金占比 | 评级 | 建议 |
|----------|------|------|
| < 10% | ⚠️ 过满 | 仓位过重，建议减仓或止盈部分持仓释放现金 |
| 10-20% | ✅ 合理 | 保持适度现金应对波动 |
| 20-35% | ⚡ 偏高 | 现金闲置，可考虑建仓观察名单标的 |
| > 35% | ⚠️ 空仓 | 现金过多，考虑加仓或配置货币基金 |

**建议操作**：
- 现金 < 10% → 列出可止盈的持仓，建议减仓释放现金
- 现金 > 30% → 列出观察名单中技术面较好的标的，建议分批建仓

#### C. 新建仓建议

**单笔建仓金额**（固定比例法）：
```
单笔风险预算 = 总组合价值 × 1-2%
建仓金额 = 单笔风险预算 / 止损比例

示例：组合 $50,000，单笔风险 1% = $500
如果止损设 5%，则建仓金额 = $500 / 5% = $10,000（即 20% 仓位）
```

**分批建仓计划**：
```
首次建仓：计划仓位的 1/3
确认加仓：技术面确认后加 1/3
回调补仓：回调至支撑位后补最后 1/3
```

**期权建仓限制**：
- 单笔期权权利金 ≤ 总组合价值的 5%
- 单标的最多 4 张合约
- 到期日 ≥ 30 天（避免短期 Theta 损耗）

#### D. 再平衡建议

**触发条件**：
- 任一持仓偏离目标仓位 > 5 个百分点
- 每季度定期再平衡
- 重大事件后（财报、拆股等）

**再平衡操作**：
```
目标仓位 = 各持仓目标占比 × 总组合价值
调仓量 = 目标仓位 - 当前仓位

正数 → 加仓
负数 → 减仓
```

**输出示例**：
```
【仓位管理】 综合评级: {健康度}
┌─────────────────────────────────────────
│ 组合总值: ${totalValue}
│ 现金: ${cash} ({cashPct}%) — {现金评级}
│ 持仓集中度: 最高 {topTicker} {topPct}% — {集中度评级}
│ 行业分布: {sectorBreakdown}
│ 同质化标的: {correlatedPairs}
├─────────────────────────────────────────
│ 📋 调仓建议:
│ {ticker1}: 当前 {currentPct1}% → 目标 {targetPct1}% ({action1} {amount1})
│ {ticker2}: 当前 {currentPct2}% → 目标 {targetPct2}% ({action2} {amount2})
│ 现金: 当前 {cashPct}% → 目标 {targetCash}%
└─────────────────────────────────────────
```

#### E. 风险预算分配

**组合层面风险**：
```
最大回撤容忍 = 总组合价值 × 10%
单只最大亏损 = 总组合价值 × 2%
期权最大亏损 = 权利金归零
```

**风险分配**：
- 股票类：总风险预算的 60-70%
- 期权类：总风险预算的 20-30%
- 现金/固收：总风险预算的 10-20%

**相关性调整**：
- 高相关持仓（ρ > 0.7）：共享风险预算
- 低相关持仓（ρ < 0.3）：独立风险预算
- 负相关持仓（ρ < 0）：可对冲，降低总风险预算占用

### 6. 市场环境判断（大盘择时）

**在分析个股前，先判断大盘环境**。通过 WebSearch 获取以下数据：

| 指标 | 数据来源 | 判断逻辑 |
|------|----------|----------|
| **S&P 500 趋势** | WebSearch "S&P 500 price vs 200 day moving average" | 价格 > MA200 → 牛市；价格 < MA200 → 熊市 |
| **VIX 恐慌指数** | WebSearch "VIX index current value" | <15 低波动（卖期权）；>30 高恐慌（买期权/抄底） |
| **Put/Call Ratio** | WebSearch "CBOE put call ratio" | >1.0 恐惧（反向做多）；<0.7 贪婪（注意回调） |
| **市场宽度** | WebSearch "NYSE advance decline ratio" | 上涨 > 下跌 → 健康；上涨 < 下跌 → 弱势 |

**市场环境评级**：

```
牛市环境: S&P500 > MA200 + VIX < 20 + 市场宽度正
  → 操作方向: 偏多，可加仓，期权买方有利

震荡环境: S&P500 在 MA200 附近 + VIX 15-25
  → 操作方向: 中性，区间操作，期权卖方策略

熊市环境: S&P500 < MA200 + VIX > 25 + 市场宽度负
  → 操作方向: 偏空，控制仓位，保护性策略
```

**输出**：
```
【市场环境】 {评级}
S&P500: {price} ({vs_ma200}) | VIX: {vix} | P/C: {pc_ratio}
操作建议: {市场环境下的操作方向}
```

### 7. 扩展技术指标

在原有 MACD/KDJ/布林带/量比基础上，增加以下指标：

| 指标 | 参数 | 计算 | 判断逻辑 |
|------|------|------|----------|
| **RSI** | 14 | RSI = 100 - 100/(1+RS), RS=avg_gain/avg_loss | >70 超买, <30 超卖; 背离检测 |
| **均线系统** | MA5/10/20/60/120/250 | 简单移动平均 | 多头排列(MA5>10>20>60)→强多；空头排列→弱 |
| **VWAP** | 日内 | 累计(价格×量)/累计量 | 价格>VWAP→日内偏多；<VWAP→偏空 |
| **斐波那契回撤** | 38.2%/50%/61.8% | 从近期高低点计算 | 回撤至38.2%→浅回调；61.8%→深回调 |
| **OBV (能量潮)** | — | 上涨日累加量，下跌日减 | OBV创新高+价格未创新高→顶背离 |
| **ATR (14)** | 14日 | TR平均值 | 动态止损 = 2×ATR；波动率度量 |

**RSI 背离检测**：
```
顶背离: 价格创新高 + RSI未创新高 → 看空信号
底背离: 价格创新低 + RSI未创新低 → 看多信号
```

**均线排列判断**：
```
多头排列: MA5 > MA10 > MA20 > MA60 > MA120
  → 强趋势，持有或加仓
空头排列: MA5 < MA10 < MA20 < MA60 < MA120
  → 弱趋势，减仓或观望
缠绕: 均线交叉密集
  → 震荡市，高抛低吸
```

### 8. 财报日历与事件驱动

**财报前（14-3天）**：
```
IV 变化: 通常提前 14 天开始上升
操作:
  - 已持仓看涨期权 → 持有等待 IV 膨胀
  - 未建仓 → 不新开期权（IV 已高）
  - 股票持仓 → 可设保护性 Put
```

**财报前（3-0天）**：
```
IV 变化: 接近峰值
操作:
  - 期权买方 → 不建议新开（IV Crush 风险）
  - 期权卖方 → 可考虑卖跨式/宽跨式
  - 股票 → 降低仓位或设好止损
```

**财报后（0-3天）**：
```
IV Crush: IV 通常下降 20-50%
操作:
  - 期权买方 → 如盈利建议止盈（IV Crush 会侵蚀价值）
  - 期权卖方 → 如盈利建议平仓（锁定利润）
  - 根据财报结果调整目标仓位
```

**其他重要事件**：
```
拆股公告: 短期利好，通常上涨 5-10%
派息日: 除息日股价自动下调股息金额
FDA 审批: 生物医药股重大事件，波动可达 20-50%
FOMC 会议: 加息/降息影响全市场，VIX 通常上升
股票回购: 长期利好，减少流通股
```

**事件日历查询**：
```
WebSearch: "{ticker} earnings date 2026"
WebSearch: "{ticker} upcoming events catalyst"
```

### 9. 期权策略推荐

根据市场判断和 IV 水平自动推荐策略：

| 市场判断 | IV 分位 | 推荐策略 | 具体操作 |
|----------|---------|----------|----------|
| 看多 | <30 | 买 Call | 买入 ATM Call，到期 30-60 天 |
| 看多 | >70 | 卖 Put | 卖出 OTM Put，收取权利金 |
| 看多 | 30-70 | 牛市价差 | 买 ATM Call + 卖 OTM Call |
| 震荡 | >70 | Iron Condor | 卖 OTM Call + 卖 OTM Put，买更远 OTM 保护 |
| 震荡 | <30 | 日历价差 | 卖近月 ATM + 买远月 ATM |
| 看空 | >70 | 买 Put | 买入 ATM Put，到期 30-60 天 |
| 看空 | 30-70 | 熊市价差 | 买 ATM Put + 卖 OTM Put |
| 保护 | — | Collar | 持有股票 + 买 OTM Put + 卖 OTM Call |
| 保护 | — | Protective Put | 持有股票 + 买 ATM/OTM Put |

**策略选择逻辑**：
```
1. 确定市场方向（看多/震荡/看空）
2. 确定 IV 分位（低/中/高）
3. 交叉匹配 → 推荐策略
4. 检查资金是否足够执行策略
5. 计算最大亏损和最大收益
```

**输出**：
```
【期权策略推荐】 {策略名称}
方向: {看多/震荡/看空} | IV分位: {iv_percentile}%
操作: {具体买卖腿}
最大收益: ${max_profit} ({max_profit_pct}%)
最大亏损: ${max_loss} ({max_loss_pct}%)
盈亏平衡: ${breakeven}
建议到期日: {expiry} ({days}天)
```

### 10. 交易心理与纪律检查

**自动检测规则**：

```
【频繁交易检测】
30天内交易次数 > 20:
  → "交易过于频繁，建议减少操作频率，提高交易质量"

【连续止损检测】
连续 3 次止损后开新仓:
  → "连续止损后建议休息 1-2 天，避免情绪化交易"

【向下摊平检测】
亏损持仓加仓（均价降低）:
  → "不建议向下摊平成本，除非有明确的加仓理由和技术支撑"

【追涨检测】
股票已上涨 > 10% 后首次买入:
  → "可能追涨，建议等待回调至支撑位再考虑入场"

【恐慌卖出检测】
单日下跌 > 5% 后立即卖出:
  → "可能恐慌卖出，建议先评估基本面是否变化"
```

**交易前检查清单**（每次开新仓时提示）：
```
□ 本次交易是否符合你的交易系统？
□ 是否有明确的入场理由（技术面/基本面）？
□ 是否设好了止盈和止损？
□ 单笔风险是否控制在 2% 以内？
□ 是否有明确的退出计划（时间/价格触发）？
□ 是否在情绪稳定状态下做出的决策？
```

### 11. 绩效追踪

**需要记录的交易数据**：
```json
{
  "trades": [
    {
      "ticker": "AAPL",
      "type": "stock",
      "entryDate": "2024-01-15",
      "entryPrice": 180.50,
      "exitDate": "2024-02-10",
      "exitPrice": 195.30,
      "shares": 100,
      "pnl": 1480,
      "pnlPct": 8.2,
      "holdDays": 26,
      "reason": "MACD金叉买入，止盈卖出"
    }
  ]
}
```

**自动计算指标**：
```
胜率: 盈利交易数 / 总交易数 × 100%
盈亏比: 平均盈利金额 / 平均亏损金额
最大回撤: (峰值 - 谷值) / 峰值 × 100%
夏普比率: (收益率 - 无风险利率) / 收益率标准差
平均持仓天数: 所有持仓天数平均值
利润因子: 总盈利 / 总亏损

最赚钱标的: PnL 最高的交易
最亏钱标的: PnL 最低的交易
最佳策略: 胜率最高的交易类型
最差策略: 亏损最多的交易类型
```

**输出**：
```
【绩效统计】 近30笔交易
胜率: {winRate}% | 盈亏比: {profitFactor}
```

### 13. 对冲与保护

**Beta 对冲**：
```
组合 Beta = Σ(持仓权重 × 个股Beta)

如果 Beta > 1.3:
  对冲比例 = (Beta - 1) × 组合价值
  对冲工具 = SPY Put (Delta ≈ -0.5)
  需要合约数 = 对冲比例 / (SPY价格 × 100 × 0.5)

如果 Beta < 0.7:
  组合偏防御，可适度增加进攻性持仓
```

**行业对冲**：
```
如果单一行业占比 > 40%:
  买入行业 ETF Put 保护
  对冲金额 = 行业持仓 × 20-30%
  
常用行业 ETF:
  科技: XLK / QQQ
  金融: XLF
  能源: XLE
  医疗: XLV
  消费: XLY / XLP
```

**VIX 对冲（黑天鹅保护）**：
```
VIX < 15 时:
  买入 VIX Call (行权价 = 当前VIX + 5)
  成本: 通常很低（0.5-2.0）
  保护: VIX 飙升时获得数倍回报
  到期: 1-3个月

VIX > 30 时:
  可卖出 VIX Put 收取权利金
  行权价 = 当前VIX - 5
```

**尾部风险保护**：
```
组合价值 > $100,000:
  建议配置 1-2% 用于尾部风险保护
  工具: 深度 OTM Put (Delta -0.05 至 -0.10)
  到期: 3-6个月
  成本: 通常很低但保护效果显著
```

```
组合 Beta: {beta} | 行业集中度: {sectorConc}%
建议:
1. {hedge_action_1}

预计成本: ${cost} ({cost_pct}%)
保护效果: 最大回撤降低 {protection}%
```

## 注意事项

1. **免责声明**：每次输出末尾附加「以上分析仅供学习参考，不构成投资建议」
2. **数据延迟**：免费 API 有 15 分钟延迟，提醒用户注意
3. **期权数据限制**：如果无法获取实时期权 IV，使用 HV20 估算并明确标注
4. **消息面时效性**：WebSearch 结果可能滞后，建议结合券商实时资讯
5. **持仓更新**：分析完成后提示用户更新持仓记忆文件
6. **新模块数据**：大盘环境、季节性、对冲建议等依赖 WebSearch，网络不通时跳过并标注"未获取"
7. **绩效追踪**：需要 portfolio.json 中包含 `trades` 数组，如无记录则跳过该模块
8. **交易纪律**：需要 `trades` 记录来计算频率和连续止损，无数据时仅输出检查清单
